# Audit App â€” Bank Statement Analysis

This Streamlit app is a **second-layer accuracy checker** for bank-statement JSON outputs generated by another application.

## What it does

- Recomputes monthly totals from `transactions` and compares them with provided `monthly_summary`.
- Separates:
  - **Real mismatches** (values differ), and
  - **Schema gaps** (field is missing in provided JSON).
- Produces an exportable `audit_report.json`.

## Compatibility improvements

The checker now supports multiple bank statement shapes while keeping original parameters (`amount_tolerance`, `strict_schema`, etc.):

- Transaction date aliases such as `date`, `posting_date`, `value_date`, `txn_date`.
- Amount aliases such as `debit/credit`, `withdrawal/deposit`, or single `amount` with optional `type` (`dr/cr`, `inflow/outflow`).
- Monthly summary aliases such as `period`/`year_month` and `txn_count`.
- Numeric parsing tolerant to values like `RM 1,234.56` and parenthesized negatives `(123.45)`.
